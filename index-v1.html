Core Primitives

🔤
Type Language
Every Effector declares its interface: input, output, context. Structural subtyping means SecurityReport is automatically compatible anywhere a ReviewReport is expected — no explicit declaration needed.
input = "CodeDiff" output = "SecurityReport" context = ["GitHubCredentials"]
⛓
Composition Algebra
Four typed operators: Sequential (→), Parallel (‖), Conditional (?), Fallback (|). Every connection has a type rule. A type error at wire-up time is infinitely cheaper than a runtime failure.
A → B iff output(A) <: input(B) A | B iff output(B) <: output(A) (substitutable fallback)
🔍
Discovery Protocol
Search by interface, not by guessing package names. The registry maintains a type index alongside the name index — find every Effector whose output is structurally compatible with your next step's input.
effector-graph query \ --input CodeDiff \ --output SecurityReport # returns 3 exact + 2 structural matches

Security Audit

effector-audit treats permissions as types — mismatches between declared interface and actual behavior are detectable statically, before a single line executes.

Permissions Are Types

An Effector declaring input: CodeDiff, output: SecurityReport has no legitimate reason to declare filesystem: ["write"]. That mismatch is a security type error — detectable before execution.

  • output: Markdown, PlainText → expect network = false
  • input: RepositoryRef → expect env-read includes *TOKEN*
  • context: [Docker] → expect subprocess = true
  • output: Notification → expect network = true
📄 effector.toml — Declared permissions
# pr-guardian/effector.toml
[effector.interface]
input  = "CodeDiff"
output = "SecurityReport"

[effector.permissions]
network    = true
env-read   = ["GITHUB_TOKEN"]
filesystem = ["read"]
subprocess = true     # ← suspicious
🔬 SKILL.md — Detected behavior
# scanning SKILL.md for patterns…

## network:external ✓
curl https://api.github.com/…

## env:read ✓
Authorization: Bearer $GITHUB_TOKEN

## write:filesystem  ← UNDECLARED
curl … | tee /tmp/output.json

## subprocess — NOT FOUND
declared but no exec() detected
Critical
Prompt injection vector. Input CodeDiff.files[].path is passed unsanitized to shell at line 34. An attacker-controlled diff path could inject arbitrary commands.
High
Permission drift: filesystem.write undeclared. Detected tee /tmp/output.json but manifest only declares filesystem: ["read"]. Add "write" or remove the operation.
Medium
Interface-permission mismatch. subprocess = true declared but zero exec()/execSync()/shell invocations detected. Over-declared permissions expand attack surface unnecessarily.
Info
Package unsigned. Signing/provenance is not yet implemented in effector-audit (coming in v0.2).
Pass
No hardcoded credentials. GITHUB_TOKEN is accessed via environment variable only — never inlined in instructions.
Pass
No data exfiltration patterns. All outbound calls target api.github.com only — consistent with declared scope.

ClawHub Corpus Analysis

Every type in effector-types is grounded in data, not intuition. 13,729 skills, analyzed and clustered. The stdlib is empirical.

13,729
skills analyzed
We clustered the entire ClawHub corpus to derive the effector-types standard library — the shared vocabulary that makes type-indexed discovery and composition type-checking possible. Every type name and its frequency below is a measured fact, not a design assumption. The most common input type is String (62%) because most skills are instruction-based. The most important typed pair for composition is CodeDiff → SecurityReport (3% × 5%) — rare but high-value, and completely invisible to keyword search.
Input Type Distribution
Insight: String dominates (62%) — most skills are instruction-based. But the 3% of skills explicitly declaring CodeDiff are the highest-value composition endpoints. Rare type pairs are invisible to keyword search; type-indexed discovery surfaces them.
Output Type Distribution
Insight: Markdown (45%) = the read-and-explain majority. JSON (28%) = the automation-ready minority — and the highest-value output type for pipeline composition, since machines can consume it without parsing.
Context Requirements
Insight: GitHubCredentials (38%) is the single most important context type. Docker + Kubernetes together at 37% define the DevOps cluster — a distinct composition neighborhood.
Skill Categories
Insight: AI/ML and Utility are nearly tied — but the 8,823 "other" skills form the long tail where gap detection matters most. Those are the undiscovered type pairs.
