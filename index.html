The Problem

AI tools don't compose

Every framework defines capabilities differently. Nothing answers: "Can these two tools actually work together?" until something breaks at runtime.

🚫

No type safety

MCP tools accept and return untyped JSON. A code review tool's output might not match what a reporting tool expects as input. You won't know until it fails.

🔒

Runtime lock-in

LangChain tools only work in Python. CrewAI tools only work in CrewAI. Your capability definitions are trapped in one ecosystem.

🔍

No composition checking

Wiring tools together is manual and error-prone. There's no way to statically verify that a pipeline of tools will actually pass compatible data.

How It Works

Declare. Verify. Compile. Compose.

Effector is a sidecar manifest. It sits next to your existing tool and adds a typed interface — without changing how your tool runs.

01

Declare your interface

Define what your tool accepts, returns, and needs. Choose from 40 standard types across input, output, and context categories. Your tool keeps running exactly as before.

effector.toml 
[effector]
name = "code-review"
version = "0.1.0"

[effector.interface]
input = "CodeDiff"
output = "ReviewReport"
context = ["Repository"]

[effector.permissions]
network = false
subprocess = false
02

Validate and audit

Static analysis catches issues before deployment: malformed manifests, missing permissions, type inconsistencies, and security mismatches between declared context and permissions.

terminal 
$ npx @effectorhq/skill-lint .
  ✓ SKILL.md structure valid
  ✓ Frontmatter complete
  ✓ All sections present

$ npx @effectorhq/audit scan .
  ✓ Permissions consistent with context types
  ✓ No network access without declaration
  ✓ 0 issues found
03

Compile to any runtime

One manifest, every runtime. The compiler emits MCP tool schemas, OpenAI function calls, LangChain tool definitions, or portable JSON IR — from the same source of truth.

compile.js 
import { compile } from '@effectorhq/core/compile';

// Same manifest → different targets
const mcp    = compile(manifest, 'mcp');
const openai = compile(manifest, 'openai-agents');
const lang   = compile(manifest, 'langchain');
const ir     = compile(manifest, 'json');
04

Compose with confidence

The type checker verifies that tool pipelines are compatible before execution. CodeDiff → ReviewReport chains are validated statically. Incompatible connections are caught at build time, not runtime.

compose.js 
import { isTypeCompatible } from '@effectorhq/core/types';

// Static verification: can these chain?
isTypeCompatible('CodeDiff', 'CodeDiff');
// → true ✓

isTypeCompatible('CodeDiff', 'ReviewReport');
// → false ✗ — caught before execution
Ecosystem

Everything you need. Nothing you don't.

9 focused packages, zero external dependencies, all using Node.js built-ins only.

Core

@effectorhq/core

Shared kernel — TOML parser, SKILL.md parser, type checker, schema validator, compiler. The single source of truth.

Core

effector-types

40 standard types across input, output, and context categories. The type catalog that makes discovery and composition possible.

Tool

@effectorhq/skill-lint

Validates SKILL.md structure, frontmatter completeness, and section coverage. Catches manifest issues before they reach production.

Tool

effector-audit

Security scanner for capability manifests. Detects permission mismatches, undeclared network access, and context-permission inconsistencies.

Tool

@effectorhq/skill-mcp

Cross-runtime compiler. Reads effector.toml and emits MCP, OpenAI, LangChain, or JSON IR targets from one manifest.

Tool

@effectorhq/skill-eval

Quality scorer with 7 metrics: frontmatter, sections, types, descriptions, interface completeness, examples, and composability.

Infra

effector-compose

Pipeline builder with static type checking. Connects capabilities by type and verifies compatibility before execution.

Infra

effector-graph

Capability dependency graph. Maps type connections across registries for discovery, visualization, and analysis.
